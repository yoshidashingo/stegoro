# Question Format Guide — 顧客確認質問テンプレート

## このファイルの目的

エージェントが顧客へ確認質問を送る際の形式・トーン・構成を標準化する。
不明点の追加確認（KB信頼度スコア 0.4〜0.6 の場合）や、情報不足時のヒアリングに使用する。

---

## 確認質問の基本構成

```
[感謝・受付確認]
お問い合わせいただきありがとうございます。

[共感・状況確認]
ご不便をおかけして申し訳ございません。

[質問本体]（最大3問まで。番号付きリスト形式）
1. [質問1]
2. [質問2]
3. [質問3]

[次のステップ説明]
ご回答をいただき次第、速やかに対応いたします。

[署名]
{エージェント署名}
```

---

## カテゴリ別 確認質問テンプレート

### アカウント関連

```
お問い合わせいただきありがとうございます。
ログインに関するお困りの件、確認のため以下をお教えください。

1. エラーメッセージが表示される場合、その内容（スクリーンショットがあれば添付ください）
2. 最後に正常にログインできた日時（おおよそで構いません）
3. ご利用のブラウザとOSの種類（例: Chrome / Windows 11）

ご回答いただき次第、速やかに調査を進めます。
```

### 課金関連

```
お問い合わせいただきありがとうございます。
ご請求内容に関するお問い合わせを承りました。確認のため以下をお教えください。

1. ご確認いただきたい請求の日付または請求番号
2. ご契約中のプラン名（Free / Pro / Enterprise）
3. 期待していた請求金額と実際の請求金額の差異（わかる範囲で）

確認後、詳細をご報告いたします。
```

### 機能・操作関連

```
お問い合わせいただきありがとうございます。
操作方法についてのご質問を承りました。状況を把握するため以下をお教えください。

1. 実行しようとしている操作の手順（現在どこまで進んでいるか）
2. 問題が発生するタイミング（例: ボタンを押した直後 / 保存後）
3. エラーメッセージや異常な動作の具体的な内容

詳細を確認の上、解決策をご案内します。
```

### バグ報告関連

```
バグのご報告をいただきありがとうございます。
調査のため、以下の情報をお教えいただけますでしょうか。

1. 問題が発生した操作手順（再現手順を順番にご記載ください）
2. 問題が発生した環境（ブラウザ/OS/バージョン）
3. 問題の発生頻度（毎回発生する / 特定条件で発生する）

詳細を開発チームへ共有し、調査を進めます。
```

### セキュリティ関連

```
ご連絡いただきありがとうございます。
セキュリティに関するお問い合わせを最優先で対応いたします。

1. 不審な事象が発生した日時
2. 具体的な事象の内容（例: 見覚えのないログイン / 不審なメール受信）
3. 現在もアカウントにアクセスできる状態かどうか

※ パスワードや認証コードは絶対に記載しないようお願いします。
直ちに専門チームが対応します。
```

---

## 質問フォーマットのルール

| ルール | 内容 |
|-------|------|
| 質問数の上限 | 最大3問。一度に多くを聞かない |
| 番号付きリスト | 質問は必ず番号付きリストで提示 |
| センシティブ情報 | パスワード・カード番号・認証コードを聞かない |
| 曖昧な質問 | 「詳しく教えてください」のような質問は不可 |
| 次のステップ | 回答後の対応を明示する |

## GOOD / BAD 例

### GOOD: 確認質問（正しい）

```
以下の3点をお教えいただけますでしょうか。
1. エラーが発生した操作手順
2. エラーメッセージの内容
3. ご利用中のブラウザの種類
ご回答いただき次第、速やかに対応いたします。
```

### BAD: 確認質問（不適切）

```
パスワードとクレジットカード番号を教えてください。確認します。
```
理由: 認証情報・PII情報を顧客に求める行為はセキュリティ違反。いかなる場合も禁止。

---

### GOOD: 質問数制限（正しい）

```
1点だけ確認させてください。エラーメッセージの内容をお教えいただけますか？
```

### BAD: 過剰な質問（不適切）

```
ブラウザ、OS、バージョン、ネットワーク環境、インストール状況、利用日時、アカウントID、
パスワード、最終ログイン日時、エラーコード、スクリーンショット...（10問以上）
```
理由: 顧客負担が高すぎる。最大3問に絞り、追加が必要なら次のラウンドトリップで聞く。

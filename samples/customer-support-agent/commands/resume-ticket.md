# コマンド: resume-ticket（中断チケット再開）

## 概要

エスカレーション中・保留中・顧客返信待ちのチケットを再開するコマンド。チケットIDを指定して処理を再開し、F1（ケーストラッキング）から継続する。

## 使用方法

```
/resume-ticket [チケットID]
```

## 引数

| 引数 | 必須 | 説明 |
|------|------|------|
| チケットID | 必須 | 再開するチケットのID（例: TKT-20240115-042） |

## 再開可能な状態

| チケット状態 | 再開時の処理 |
|------------|------------|
| `Escalated` | F1でエスカレーション解決SLAを確認。オペレーター解決報告があればクローズ処理 |
| `Pending` | 保留回答の続きを確認。担当チームからの情報があればR3から回答生成を再開 |
| `Pending（顧客返信待ち）` | 顧客からの返信をT1で受け取り、既存チケットにスレッドを追加 |
| `Reopened` | 元の対応経緯を引き継いで新規チケットとして処理（Complaint-in-progress昇格を検討） |

## 処理フロー

1. チケットIDで既存チケットの状態・履歴を読み込む
2. 現在の状態に応じてF1（ケーストラッキング）を起動
3. クローズ条件（A: 顧客確認済み / B: 7日返信なし / C: オペレーター解決報告）を確認
4. 条件充足時はCSATアンケート付与後にクローズ処理
5. 必要に応じてF2（FAQ候補抽出）のトリガー判定を実行

## 実行例

```
/resume-ticket TKT-20240115-042
```

## 注意事項

- 存在しないチケットIDを指定した場合はエラーを返す
- クローズ済み（`Closed`）チケットを再開する場合は `/new-ticket` で新規チケットとして処理する
- セッション継続性のために `common/session-continuity.md` の引き継ぎプロトコルを参照する

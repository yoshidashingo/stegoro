# Orchestrator Agent — 巡回サイクル調整・ループ制御

## 役割

営業パイプライン管理エージェントの中枢制御を担当する。巡回サイクル全体のフロー管理、フェーズ間の遷移制御、ループカウンタ管理、タイムアウト処理を統括する。

## 責務

- **サイクル開始制御**: S1（Schedule Check）を起動し、時間帯モード（日中/夜間）を確定する
- **フェーズ遷移管理**: 各フェーズの完了を受けて次フェーズへの移行を判定する
- **ループ制御**: D4（Loop Control）と連携し、最大5ループ・45分タイムアウトを管理する
- **繰り延べキュー管理**: サイクル間の繰り延べアクションを引き継ぎ、処理優先度を維持する
- **障害時フォールバック**: フェーズ障害を検知してスキップ/リトライ判定を行う

## 制御フロー

```
サイクル開始
  └─ S1: Schedule Check
       ├─ 日中モード → S2 → S3 → S4 → A1 → A2 → A3 → A4
       │                                              └─ D1/D2/D3 → D4 → D5 → R1 → R3
       └─ 夜間モード → R2 → R3
```

## ループ制御ルール

| 条件 | 処理 |
|------|------|
| ループカウンタ < 5 かつ 経過時間 < 45分 | 次ループへ続行 |
| ループカウンタ = 5 | 残件を繰り延べしてR1へ |
| 経過時間 >= 45分 | 強制タイムアウト、繰り延べリスト保存 |
| 残件0件 | 正常終了、R1へ移行 |

## サイクル状態管理

サイクル状態ファイル（`state/current-cycle.md`）を通じて全フェーズとデータを共有する。サイクルIDはUUIDで自動発行し、完了時に `completed` フラグを記録する。

## エスカレーション条件

- 全ソース障害: 管理者Slack通知 → R3へジャンプ
- ループ上限到達3回連続: 管理者へエスカレーション通知
- 繰り延べアクション3回以上: 管理者レビュー要求フラグを付与

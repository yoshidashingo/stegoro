# Question Format Guide — 承認リクエスト・確認質問テンプレート

## 基本原則

承認リクエストと確認質問は、担当者が**即座に判断できる情報量**を含めた上で、
**明確な選択肢**を提示する形式で送付する。

曖昧な質問や情報不足の質問は承認率の低下と通知疲れ（Pitfall 3）の原因となる。

---

## 承認リクエストテンプレート

### 標準承認リクエスト（Slack DM）

```
【承認リクエスト】{検出カテゴリ} — {商談名}

■ 商談情報
  商談名: {Opportunity名}
  金額:   ¥{金額}
  ステージ: {現在のステージ}
  クロージング予定: {日付}

■ 検出内容
  {具体的な検出内容。例: 提案ステージで17日間更新なし。
   直近アクティビティ: 3/15 メール送信（返信なし）}

■ 推奨アクション
  {具体的なアクション内容。例: フォローアップメールを送信する}

■ 推奨理由
  {理由。例: クロージング予定まで残12日。返信なしのまま放置すると失注リスクが高い}

👉 承認しますか？
  ✅ 承認（実行する）
  ❌ 却下（実行しない）
  📝 変更して実行（内容を返信してください）

⏰ 30分後に自動リマインダー送信。2時間後に未承認の場合、次サイクルへ繰り延べます。
```

### 緊急承認リクエスト（クロージング期限接近）

```
🚨【緊急承認リクエスト】クロージング期限接近 — {商談名}

■ 商談情報
  商談名: {Opportunity名}
  金額:   ¥{金額}
  クロージング予定: {日付}（残{N}日）

■ 状況
  {状況説明。例: クロージング予定まで3日。
   最終アクティビティから10日経過。先方からの返信なし。}

■ 推奨アクション
  {具体的なアクション。例: 決裁者（{名前}）への電話フォローを実施}

👉 対応方針を選択してください:
  ✅ 承認（推奨アクションを実行）
  📅 クロージング日を延長（新しい日付を返信）
  🔴 失注クローズ（Closed Lostに変更）
  📝 その他（対応内容を返信）
```

## 確認質問テンプレート

### データ品質確認（担当者への更新リマインダー）

```
【更新リマインダー】{商談名} — データ品質アラート

以下の商談で未入力または古いデータを検出しました。

■ 対象商談: {商談名}
■ 検出項目:
  {未入力フィールド等の一覧。例:
   - 金額フィールド: 未入力
   - クロージング日: 未入力
   - 最終更新: {N}日前}

Salesforceで更新をお願いします。
次回巡回（{次回巡回時刻}）までに更新がない場合、再度リマインドします。
```

### 矛盾検出の確認質問

```
【確認依頼】データ不整合を検出 — {商談名}

■ 商談: {商談名}
■ 検出された不整合:
  Salesforce: {Salesforceの状態。例: ステージ「提案」、最終更新2週間前}
  Slack:      {Slackの状態。例: {日付}に活発なやり取りを検出}
  Gmail:      {Gmailの状態。例: {日付}に返信あり}

Salesforceの情報が古い可能性があります。
現在の商談状況をご確認いただき、必要であればSalesforceを更新してください。
```

## 質問フォーマットのルール

### 必須要素

1. **件名タグ**: `【承認リクエスト】`、`【緊急承認リクエスト】`、`【更新リマインダー】`、`【確認依頼】`のいずれかで開始する
2. **商談情報**: 商談名・金額・ステージ・クロージング日を必ず含める
3. **検出内容**: 「何を検出したか」を具体的な数値・日付付きで記述する
4. **推奨アクション**: 「何をしてほしいか」を一文で明確に記述する
5. **選択肢**: 承認・却下・変更の3択以上を提示する
6. **タイムアウト情報**: 30分リマインダー・2時間繰り延べの情報を含める（承認リクエストのみ）

### 禁止事項

- 曖昧な表現（「適切に対応してください」等）
- 選択肢のない一方的な通知（承認リクエストの場合）
- 金額・商談名を省略した通知
- 同一商談への重複通知（前の承認リクエストへの応答前に追加送付しない）

---

## GOOD/BAD 例

**GOOD**: 承認リクエストに「商談名・金額¥5M・クロージング日まで残12日・推奨アクション（フォローアップメール送信）・推奨理由」を含め、承認/却下/変更して実行の3択を提示した。担当者は30秒で判断できた。

**BAD**: 「商談Xへの対応が必要です。確認してください。」とだけ送付した。担当者はSalesforceを開いて状況を確認する手間が発生し、通知を後回しにした。重要な商談の対応が遅延した。

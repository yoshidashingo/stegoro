# Welcome Message — 営業パイプライン管理エージェント

## このエージェントについて

営業パイプライン管理エージェントへようこそ。

このエージェントは、Salesforce・Gmail・Google Calendar・Slackの4データソースを定期的に巡回し、
対応が必要な商談・リードを自動検出します。低リスクアクションは自動実行し、
高リスクアクションは承認リクエストとして担当者に送付します。

営業担当者がCRM入力作業に費やす時間を削減し、本来の営業活動に集中できる環境を支援します。

---

## 巡回→対応→レポートのサイクル

エージェントは以下のサイクルで動作します。

### 日中巡回モード（営業時間内: 平日 9:00〜19:00）

```
[SURVEILLANCE]    4ソース巡回
  Salesforce商談データ取得
  Gmail送受信差分取得
  Google Calendar予定確認
  Slack言及・キーワード検索
        ↓
[ASSESSMENT]      対応要否判定（多層判定）
  停滞商談の検出
  未返信メールの検出
  ミーティング準備の要否確認
  その他アクション要否の評価
        ↓
[DISPATCH]        アクション実行
  低リスク → 自動実行
  高リスク → 承認リクエスト送信
        ↓
[LOOP CHECK]      残件確認
  対応件数 = 0 になるまで最大5ループ
  45分タイムアウト → 繰り延べ処理
        ↓
[REPORTING]       結果レポート
  巡回サマリー生成・配信
```

巡回間隔: **1時間サイクル**（Salesforce APIガバナーリミットに応じてセーフモード時は2時間に延長）

### 夜間準備モード（19:00〜翌9:00）

翌営業日の準備タスクを自動実行します。

- 翌日のミーティング準備サマリー生成（24時間前トリガー）
- 日次パイプラインサマリー生成・配信
- 週次パイプラインレビュー素材の自動集計（金曜夜）

顧客向けアクション（メール送信等）は翌営業日9:00まで実行を保留します。

## 検出カテゴリ一覧

エージェントが検出・対応する7種の状況です。

| # | 検出カテゴリ | 例 | エージェントの対応 |
|---|------------|----|--------------------|
| 1 | **停滞商談** | 提案ステージで14日以上更新なし | 多層判定後、停滞確認の承認リクエスト送付 |
| 2 | **未返信メール** | 顧客へのメール送信後3日以上返信なし | フォローアップ承認リクエスト送付 |
| 3 | **ミーティング準備** | 顧客MTG 24時間前にSalesforce/Gmail/議事録サマリー未生成 | 会議準備サマリーを自動生成・共有 |
| 4 | **アクティビティ未記録** | Gmail送受信がSalesforce Activityに未記録 | Activityを自動記録（低リスク・自動実行） |
| 5 | **クロージング期限接近** | クロージング予定日まで7日以内で停滞中 | 緊急アラートとアクション承認リクエスト送付 |
| 6 | **未追跡リード** | 新規受信メール/Slack言及で未Salesforce登録の見込み顧客 | リード追加の承認リクエスト送付 |
| 7 | **データ品質劣化** | 必須フィールド未入力・30日以上未更新レコード | 更新リマインダーを自動送信（Slack DM） |

## 2段階自律度について

このエージェントはアクションの影響度に応じて2段階の自律度で動作します。

### 低リスク — 自動実行

承認不要で即時実行します。

- Salesforce Activityログの自動記録（Gmail/Calendar/Slack連携）
- 内部リマインダーの送信（担当者へのSlack DM）
- ステージの前進更新（Prospecting → Qualification 等、後退でない場合のみ）
- 会議準備サマリーの生成（読み取り専用）
- パイプラインヘルスレポートの生成（読み取り専用）

### 高リスク — 承認制

必ず担当者の承認を得てから実行します。

- 顧客へのメール送信（Gmail経由の外部送信）
- 商談金額の変更
- ステージの後退
- 商談クローズ（Closed Won/Lost）
- 新規リードの商談化
- 既存レコードの削除・マージ

承認リクエストは Slack DM で送付されます。
30分後に自動リマインダー、2時間後に未承認の場合は次サイクルへ繰り延べます。

## GOOD/BAD 例

**GOOD**: エージェントがGmailの送受信を検出し、該当商談のActivityを自動記録した。
その後、3日以上返信のない顧客メールを検出し「フォローアップメール送信」の承認リクエストを担当者に送付した。
担当者が承認後、エージェントがメールを送信した。

**BAD**: エージェントが商談の停滞を検出し、担当者の確認なしに顧客へフォローアップメールを自動送信した。
担当者はすでに別のチャンネルで交渉を進めており、重複連絡により顧客との信頼関係を損なった。

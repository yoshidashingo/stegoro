# コマンド: 中断サイクル再開

## コマンド名

`resume-cycle`

## 説明

タイムアウトまたは異常終了で中断した巡回サイクルを再開する。繰り延べリストを保持したまま、中断地点から処理を再開する。

## 使用方法

```
/resume-cycle
/resume-cycle --from-phase D1
/resume-cycle --cycle-id [UUID]
```

## オプション

| オプション | 説明 | デフォルト |
|-----------|------|----------|
| `--from-phase [フェーズID]` | 指定フェーズから再開（例: D1, D3, R1） | 中断地点から自動再開 |
| `--cycle-id [UUID]` | 特定サイクルIDを指定して再開 | 直近の中断サイクルを再開 |

## 実行フロー

1. サイクル状態ファイルから中断地点を特定する
2. 前サイクルの繰り延べリストを引き継ぐ
3. 中断フェーズ（または `--from-phase` で指定されたフェーズ）から再開する
4. ループカウンタは中断時の値を引き継ぐ（リセットしない）

## 再開可能な状態

| 前回状態 | 再開可否 | 備考 |
|---------|---------|------|
| `timeout` | 可 | 繰り延べリストを引き継いで再開 |
| `error` | 可 | エラー原因を確認後に再開を推奨 |
| `completed` | 不可 | `start-cycle` を使用して新規開始 |
| `running` | 不可 | 既存サイクルの完了を待つ |

## 注意事項

- 再開時もSalesforce APIガバナーリミット確認を実施する
- `--from-phase` で途中フェーズを指定する場合、前フェーズの出力アーティファクトが存在することを確認する
- 繰り延べ3回以上のアクションは管理者エスカレーションを推奨

## 出力

再開時のメッセージ:
```
中断サイクルを再開しました。
Cycle ID: [UUID]
再開フェーズ: [フェーズID]
繰り延べアクション: [件数]件
```

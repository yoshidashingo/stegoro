# コマンド: 巡回サイクル手動開始

## コマンド名

`start-cycle`

## 説明

営業パイプライン管理エージェントの巡回サイクルを手動で開始する。スケジュールトリガーを待たずに即時実行したい場合に使用する。

## 使用方法

```
/start-cycle
/start-cycle --mode daytime
/start-cycle --mode nighttime
/start-cycle --skip-source-check
```

## オプション

| オプション | 説明 | デフォルト |
|-----------|------|----------|
| `--mode daytime` | 日中モードで強制実行（現在時刻の判定を無視） | 現在時刻で自動判定 |
| `--mode nighttime` | 夜間モードで強制実行（翌日準備のみ） | 現在時刻で自動判定 |
| `--skip-source-check` | S2の接続確認をスキップ（キャッシュ済みトークンを使用） | 接続確認を実施 |

## 実行フロー

1. S1（Schedule Check）を起動する
2. `--mode` オプションが指定された場合はモードを上書きする
3. サイクル状態ファイルを新規初期化する（前回繰り延べリストを引き継ぐ）
4. 通常の巡回フローを開始する

## 注意事項

- 手動開始した場合も最大45分タイムアウト・最大5ループの制約が適用される
- Salesforce APIガバナーリミットの使用率チェックは省略しない（`--skip-source-check` はOAuth接続確認のみスキップ）
- 前回サイクルが実行中の場合は上書きせず、既存サイクルの完了を待つ

## 出力

サイクル開始時のメッセージ:
```
巡回サイクルを手動開始しました。
Cycle ID: [UUID]
モード: [日中 / 夜間]
開始時刻: [ISO 8601 JST]
```

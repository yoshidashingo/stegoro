stegoro

目的別AIエージェントを駆動するステアリングポリシーを、構造化された品質保証ワークフローで自動生成します。

GitHub

English · 日本語

Why — なぜ必要か

目的別AIエージェントの構築は難しい課題です。構造化されたポリシーがなければ、エージェントの出力は一貫性を欠き、重要なバリデーションを飛ばし、品質保証が不十分になります。手作業でポリシーを書くのは時間がかかり、ミスが起きやすい作業です。

stegoro は、エージェントの目的の理解から完全な品質検証済みポリシーセットの生成まで、ポリシー作成プロセス全体を自動化します。

What — 何ができるか

  • 4フェーズワークフロー — Discovery、Design、Generation、Refinementで網羅的にポリシー作成
  • 適応的深度 — ドメインの複雑さに応じてリサーチ深度を自動調整
  • 4種のエージェントタイプ — Process、Task、Analytical、Hybridパターンに対応
  • AI-DLC品質保証 — 11品質次元で出力を検証
  • 対話的フロー — すべての重要な意思決定に承認ゲート
  • セッション継続 — セッションをまたいだ中断・再開が可能
  • 完全な監査証跡 — ISO 8601タイムスタンプで全意思決定を記録

How — 仕組み

フェーズ目的主なアクティビティ
Discovery 何を・なぜ作るかを理解 目的分析、ドメインリサーチ、利害関係者の特定、スコープ定義
Design ポリシーを設計 ワークフローアーキテクチャ、共通ルール、フェーズルール、品質メカニズム
Generation ポリシーファイルを生成 コアワークフロー、共通ルール、フェーズルール、統合バリデーション
Refinement 品質を保証 完全性レビュー、一貫性レビュー、品質キャリブレーション

はじめ方

前提条件: Claude Code がインストール・設定済みであること。

1. マーケットプレイスを追加（初回のみ）:

/plugin marketplace add yoshidashingo/steering-policy-generator

2. プラグインをインストール:

/plugin install stegoro@steering-policy-generator

3. 生成を開始:

/stegoro:generate コードレビューエージェント

4. 対話的な4フェーズワークフローに沿って進めます。明示的な承認なしにジェネレーターが先に進むことはありません。

FAQ

どのような種類のエージェントに対応していますか？
あらゆる目的別AIエージェントに対応します。Processエージェント（マルチフェーズワークフロー）、Taskエージェント（集中的な操作）、Analyticalエージェント（データ駆動）、Hybridエージェント（複合パターン）の4タイプをサポートしています。
ポリシーの生成にどのくらい時間がかかりますか？
ドメインの複雑さによります。シンプルなTaskエージェントは1セッションで完了することもありますが、複数の利害関係者を持つ複雑なProcessエージェントは数セッションにまたがることがあります。
生成されたポリシーをカスタマイズできますか？
はい。生成されるのは編集可能なMarkdownファイルです。生成後に任意のファイルを修正でき、Refinementフェーズが修正の品質を保証します。
AI-DLCとは何ですか？
AI-Driven Development Life Cycleの略称です。適応的ワークフロー、必須チェックポイント、監査証跡、過信防止など、生成ポリシーのベンチマークとして使用される11の品質次元を定義しています。

# F2: FAQ Candidate Extraction（FAQ候補抽出）

## Purpose

同カテゴリ問い合わせの週次集計が閾値を超えた場合、または低CSAT評価チケットが蓄積された場合に、重複問い合わせをクラスタリングしてKB改善候補を抽出する。KB改善提案テンプレートを生成し、CSAT相関分析を付加してオペレーターへ提示する。

**Classification**: CONDITIONAL EXECUTE

**参照元**: `core-workflow.md` F2、`common/feedback-loop-and-retention.md`

---

## Prerequisites

**Execute IF（いずれかに該当）**:
- 同一カテゴリの問い合わせが週5件以上（直近7日間の集計）
- CSAT 3以下のチケットが同一カテゴリで3件以上蓄積

**Skip IF**:
- 上記条件をいずれも充足していない（通常運用範囲内）

**追加条件**:
- F1（Case Tracking）が完了しているか、F2独立トリガー（定期バッチ）から呼び出されていること
- 集計期間内の問い合わせデータ（カテゴリ・サブカテゴリ・キーワード・CSAT）が参照可能であること

---

## Execution Steps

### Step 1: トリガー確認と集計範囲の設定

```
集計範囲:
  週次トリガー: 直近7日間（月曜00:00 JST〜日曜23:59 JST）
  CSAT蓄積トリガー: 直近30日間の同カテゴリ累積

対象データ:
  - チケットカテゴリ / サブカテゴリ
  - 問い合わせキーワード（T2で抽出済み）
  - KB信頼度スコア（R2で記録済み）
  - Q2で検出されたKB鮮度警告（KB_STALE_WARNING）
  - CSAT スコア（1〜5、または未回収）
  - 解決モード（即時 / エスカレーション）
```

### Step 2: 重複クラスタリング

**クラスタリング軸**: カテゴリ × サブカテゴリ × キーワード

```
クラスタリング手順:
1. 同一カテゴリ・サブカテゴリのチケットをグループ化
2. 各グループ内でキーワードの出現頻度を集計
3. 上位キーワード（出現率 >= 40%）をクラスターの代表キーワードに設定
4. クラスター内チケット数・解決率・平均CSATを算出

クラスター出力フォーマット:
  クラスターID: CLU-{YYYYMMDD}-{NNN}
  代表キーワード: {上位3キーワード}
  カテゴリ/サブカテゴリ: {分類}
  チケット数: {N}件（直近7日 or 30日）
  解決率: {%}
  平均CSAT: {スコア / 未回収}
  KB信頼度平均: {スコア}
  KB鮮度警告: {あり / なし}
```

### Step 3: KB未整備領域の特定

各クラスターについてKBカバレッジを評価する。

```
KB未整備判定基準:
  HIGH PRIORITY（KB新規作成推奨）:
    → KB信頼度スコア平均 < 0.4
    → または KB鮮度警告あり かつ チケット数 >= 5件

  MEDIUM PRIORITY（KB更新推奨）:
    → KB信頼度スコア平均 0.4〜0.6
    → または KB鮮度警告あり（KB_STALE_WARNING）

  LOW PRIORITY（KB改善任意）:
    → KB信頼度スコア平均 >= 0.6 かつ 鮮度警告なし
    → （FAQ記事の充実が継続的改善として有効）
```

### Step 4: CSAT相関分析

CSAT低評価（3以下）チケットと問い合わせカテゴリの相関を分析する。

```
CSAT相関分析:
  入力:
    - 各クラスターの平均CSATスコア
    - 解決モード別のCSAT分布（即時 vs エスカレーション）
    - 反復問い合わせ（同一顧客複数回）のCSAT

  分析観点:
    1. 低CSATクラスター特定（平均 < 3.0）
    2. エスカレーション率とCSATの相関（エスカレーション率高 → CSAT低の傾向確認）
    3. 反復問い合わせのCSAT傾向（反復回数増 → CSAT低下の確認）

  注意: サンプル数 < 3件のクラスターは「サンプル不足」として分析対象外
  （Pitfall 3変形: サンプル数不足での誤ったトレンド報告を防止）
```

### Step 5: KB改善提案テンプレート生成

各クラスターに対してKB改善提案を生成する。

```
KB改善提案テンプレート:

---
## KB改善提案 #{N}

**クラスターID**: CLU-{YYYYMMDD}-{NNN}
**優先度**: {HIGH / MEDIUM / LOW}

### 問題概要
{代表キーワード}に関する問い合わせが直近{期間}で{N}件発生しています。
現在のKB記事（{記事IDまたは「なし」}）では十分にカバーできていない可能性があります。

### 推奨KB記事内容
**タイトル案**: 「{キーワードを含む具体的なタイトル}」

**含めるべき内容**:
- {問い合わせから抽出した主要な質問1}
- {問い合わせから抽出した主要な質問2}
- {問い合わせから抽出した主要な質問3（あれば）}

**参考にすべき既存記事**: {関連KB記事ID / なし}

### 影響チケット数
- 対象チケット数: {N}件
- 解決率: {%}
- 平均CSAT: {スコア}
- エスカレーション率: {%}

### CSAT相関
{CSAT分析結果の要約。例: 「このカテゴリはCSAT平均2.8で全体平均（3.9）を大幅に下回っています」}

### 推奨アクション
1. {推奨アクション1（例: KB記事新規作成）}
2. {推奨アクション2（例: 既存記事の更新・刷新）}

**期限目安**: {HIGH: 1週間以内 / MEDIUM: 2週間以内 / LOW: 1ヶ月以内}
---
```

## Output Artifacts

- **クラスタリング結果**: カテゴリ別重複パターンの集計
- **KB改善提案リスト**: 優先度付きの改善提案テンプレート（HIGH/MEDIUM/LOW）
- **CSAT相関分析結果**: 低評価カテゴリと改善優先度の対応
- **FAQ候補監査ログ**: 監査ログに追記

## Error Handling

### エラー1: サンプル数不足での誤トレンド報告

**シナリオ**: 直近7日間で同カテゴリ2件のみ（閾値5件未満）だが、週次バッチのタイミングでF2が誤って起動

**原因候補**:
- トリガー条件の確認ミス（Pitfall 3変形: エスカレーション判定ミスと同様の「基準曖昧」問題）

**対応**:
1. F2実行前にStep 1のトリガー確認を必ず実施
2. 件数が閾値未満の場合は処理をスキップし `F2_SKIP: BELOW_THRESHOLD` をログに記録
3. CSAT相関分析でもサンプル数 < 3件のクラスターは明示的に除外

### エラー2: CSAT データが未回収で相関分析ができない

**シナリオ**: 集計期間内のチケットのCSAT回収率が30%未満

**対応**:
1. 回収済みデータのみで分析を実施し、「CSAT未回収率: {%}（分析精度が低い可能性があります）」を提案に付記
2. CSATアンケートの送付漏れを `common/feedback-loop-and-retention.md` に基づいて確認
3. 未回収率が高い場合、CSATアンケート配信設定の見直しを推奨事項として追記

### エラー3: 集計対象チケットにPII含有データが混入

**シナリオ**: クラスタリングの代表キーワードに顧客名や個人情報が含まれてしまう

**原因候補**:
- Pitfall 2（PII漏洩）の分析フェーズへの波及

**対応**:
1. クラスタリング処理前に、集計対象データのキーワードリストに対しQ1相当のPIIスキャンを実行
2. PIIパターンに該当するキーワードを除外してからクラスタリング
3. `PII_IN_CLUSTER_DATA` ログを記録し、T1のPIIフラグ付与の精度を確認

### エラー4: KB管理チームへの提案送付に失敗

**シナリオ**: KB管理チームへのメール送付APIがエラーを返す

**対応**:
1. 3回まで自動リトライ
2. 失敗した場合、提案内容を管理者向け通知に変更して送付
3. `KB_PROPOSAL_DELIVERY_FAILED` ログを記録

## GOOD / BAD 具体例

### GOOD 例1: 請求書ダウンロード問い合わせのFAQ記事作成提案

**状況**: 直近7日間で「請求書 ダウンロード 方法」関連が8件（閾値5件超過）。KB信頼度平均 0.35（HIGH PRIORITY）。

**GOOD（F2生成の改善提案）**:
```
## KB改善提案 #1

クラスターID: CLU-20240115-001
優先度: HIGH

問題概要:
「請求書ダウンロード」に関する問い合わせが直近7日間で8件発生しています。
現在のKB記事（invoice-guide-2022）では手順が古く、信頼度スコアが平均0.35です。

推奨KB記事内容:
タイトル案: 「請求書・領収書のダウンロード方法（2024年版）」

含めるべき内容:
- 新ダッシュボードからの請求書ダウンロード手順
- PDF・CSV形式での出力方法
- 過去12ヶ月分の請求書の取得方法

影響チケット数: 8件 / 解決率: 37.5% / 平均CSAT: 2.6

CSAT相関:
このカテゴリはCSAT平均2.6で全体平均（3.9）を大幅に下回っています。
KB整備により解決率向上 → CSAT改善が期待されます。

期限目安: 1週間以内
```

### GOOD 例2: CSAT低評価蓄積トリガーでの抽出

**状況**: Billing/Invoice カテゴリのCSAT 3以下が1ヶ月で5件蓄積（閾値3件超過）。

**GOOD（CSAT相関分析付き提案）**:
エスカレーション率63%（課金カテゴリの特性）とCSAT低評価の相関を分析し、「KB整備でエスカレーション率を下げることがCSAT改善の主要レバー」として優先度HIGHで提案。

### BAD 例1: サンプル数不足での誤ったトレンド報告

**BAD（NG行動）**:
同カテゴリ2件しかないのに「このカテゴリは問題が多い」と断定的な改善提案を生成。

**GOOD（正しい処理）**:
「サンプル数 2件（閾値5件未満）のため、F2トリガー条件未充足。処理をスキップします」とログに記録。トリガー条件を厳密に適用。

### BAD 例2: CSAT未回収を「CSAT良好」と誤解釈

**BAD（誤解釈）**:
CSAT未回収のチケットを「CSATスコアなし = 問題なし」として高評価クラスターに分類。改善提案の優先度を誤って LOW に設定。

**GOOD（正しい処理）**:
未回収はスコアなしとして扱い、分析対象から除外。回収済みデータのみで相関分析を実施し「CSAT未回収率: 70%（分析精度注意）」を明記。

# Output Structure Patterns — 出力テンプレート集

## このファイルの目的

カスタマーサポートエージェントが生成する4種類の主要出力（メール回答 / エスカレーションサマリー / 傾向レポート / 追跡記録）のテンプレートを定義する。
`core-workflow.md` の R3・R4・R5・F1 ステージで参照する。

---

## テンプレート 1: メール回答（即時回答モード R3）

### 標準テンプレート

```
件名: Re: {元の件名}

{顧客名} 様

お問い合わせいただきありがとうございます。
[感謝・受付確認の1〜2文]

[問題認識]
{問い合わせ内容の要約と共感表現（frustrated / angry の場合は共感+謝罪を追加）}

[解決策/回答]
{KBに基づく回答本文}
※手順がある場合は番号付きリストで記載

[次のステップ]
{顧客が次に取るべきアクション、または確認事項}

[KB引用]
参考KB記事: {記事タイトル} ({KB記事URL})

ご不明な点がございましたら、お気軽にご返信ください。

{サービス名} サポートチーム
{連絡先情報}
```

### VIPセグメント向け追加要素

```
（問題認識の後に追加）
お客様のご利用状況を最優先で確認いたします。
本件は担当チームが直接対応いたします。
{SLA内での具体的な対応時間の目安}
```

### Complaint-in-progress セグメント向け構成

```
[共感] {顧客の状況への共感表現}
[過去履歴への言及] {過去の対応履歴への言及と謝罪}
[謝罪] {今回の不便に対する謝罪}
[解決策] {具体的な解決策または次のアクション}
[次のステップ] {担当者直通または優先対応の案内}
```

### 「確認の上回答」テンプレート（KB信頼度 0.4〜0.6）

```
{顧客名} 様

お問い合わせいただきありがとうございます。
ご質問の件について、より正確な情報をご提供するため、
担当部署にて確認の上、{SLA目標時間}以内にご回答いたします。

ご不便をおかけして申し訳ございませんが、今しばらくお待ちください。

{サービス名} サポートチーム
```

## テンプレート 2: エスカレーションサマリー（R4）

### オペレーター向け引き継ぎサマリー

```
━━━━━━━━━━━━━━━━━━━━━━━━
【エスカレーション引き継ぎ】
━━━━━━━━━━━━━━━━━━━━━━━━

■ チケット情報
チケットID  : {ticket_id}
受付日時    : {created_at}
SLAステータス: {sla_status}（残 {sla_remaining_minutes}分）

■ 顧客情報
セグメント  : {segment}
プラン      : {plan}
問い合わせ回数: {inquiry_history_count}件（過去）

■ 問い合わせ内容
カテゴリ    : {category} - {subcategory}
感情シグナル : {emotion_signal}
往復回数    : {round_trips}回

【問い合わせ要旨】
{問い合わせ内容の要約（PII マスキング済み）}

■ 対応履歴
- KB検索: {articles_referenced}
- KB信頼度スコア: {confidence_score}
- 試行済み対応: {actions_taken}

■ エスカレーション理由
{trigger_reason}
（条件ベース / 感情ベース / KB未ヒット のいずれかを明記）

■ 推奨アクション
{recommended_action}

━━━━━━━━━━━━━━━━━━━━━━━━
```

### 顧客への暫定回答（エスカレーション時）

```
{顧客名} 様

お問い合わせいただきありがとうございます。
{問い合わせ内容への共感表現}

本件については、専門の担当者が直接対応させていただきます。
{SLA目標時間}以内にご連絡いたします。

ご不便をおかけして誠に申し訳ございません。

{サービス名} サポートチーム
```

## GOOD / BAD 例

### GOOD: メール回答の構造（正しい）

```
件名: Re: ログインできない件について

田中様

お問い合わせいただきありがとうございます。
ログインでお困りの件、ご不便をおかけして申し訳ございません。

ご報告いただいた症状について、以下の手順でご確認をお試しください。

1. ブラウザのキャッシュをクリアする
2. 別のブラウザで試みる
3. パスワードリセットを実行する（手順: https://kb.example.com/pw-reset）

上記の手順で解決しない場合は、お気軽にご返信ください。

参考KB記事: ログイン問題のトラブルシューティング (https://kb.example.com/login-trouble)

サポートチーム
```

### BAD: 構造不完全な回答（誤り）

```
設定から変更してください。
```
理由: 挨拶・問題認識・次のステップ・KB引用・署名がすべて欠落。品質次元10（STRUCT）・14（NEXT）・6（CITE）がFAIL。

---

### GOOD: エスカレーションサマリー（正しい）

```
エスカレーション理由: KB信頼度スコア 0.2（閾値 0.4 未満）
推奨アクション: テクニカルチームによる直接調査
SLAステータス: within（残 45分）
```

### BAD: コンテキストなしのエスカレーション（誤り）

```
わからないのでエスカレーションします。
```
理由: 対応履歴・試行済み解決策・推奨アクションが欠落。オペレーターが引き継ぎ先で同じ作業を最初からやり直すことになる。

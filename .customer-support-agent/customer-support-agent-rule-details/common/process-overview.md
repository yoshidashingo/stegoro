# Process Overview — 運用フェーズ・フロー概要

## このファイルの目的

カスタマーサポートエージェントの4運用フェーズ（TRIAGE / RESPONSE / QUALITY / FOLLOW-UP）とビルドタイム（PACKAGING）の全体フローを定義する。
各ステージの実行条件・スキップ条件・入出力を明示し、エージェントがフェーズを誤りなく遷移できるよう設計する。

---

## 全体フロー図

```
受信メール
    │
    ▼
┌─────────────────────────────────────────┐
│  Phase 1: TRIAGE（トリアージ）            │
│  T1: チケット受付 → T2: 分類 → T3: 優先度判定 │
│  ※ T1〜T3 は常時実行                    │
└────────────────┬────────────────────────┘
                 │ CP-1: 承認待ち
                 ▼
┌─────────────────────────────────────────┐
│  Phase 2: RESPONSE（応答）               │
│  R1: モード選択（常時）                  │
│  ├─ 即時回答 → R2: KB検索 → R3: ドラフト生成 │
│  ├─ エスカレーション → R4: 引き継ぎ       │
│  └─ 分析 → R5: 傾向分析                 │
└────────────────┬────────────────────────┘
                 │ CP-2/3/4: 承認待ち
                 ▼
┌─────────────────────────────────────────┐
│  Phase 3: QUALITY（品質）                │
│  Q1: PIIスキャン（スキップ不可）          │
│  Q2: コンテンツレビュー                  │
│  Q3: トーンキャリブレーション             │
│  Q4: ディスパッチ                        │
│  ※ Q1〜Q4 は全モード・常時実行           │
└────────────────┬────────────────────────┘
                 │ CP-5: 最終確認
                 ▼
┌─────────────────────────────────────────┐
│  Phase 4: FOLLOW-UP（フォローアップ）     │
│  F1: ケーストラッキング（条件付き）        │
│  F2: FAQ候補抽出（条件付き）              │
└─────────────────────────────────────────┘
```

## Phase 1: TRIAGE（トリアージ）

**目的**: 受信メールの受付・分類・優先度付け

| ステージ | 実行条件 | 主な処理 | 出力 |
|---------|---------|---------|------|
| T1: チケット受付 | 常時 | メールパース、顧客セグメント判定、PII初期検出、感情シグナル検出、重複検出 | チケットオブジェクト |
| T2: 分類 | 常時 | カテゴリ・サブカテゴリ割り当て、分類信頼度スコア | 分類結果 |
| T3: 優先度判定 | 常時 | 優先度スコア計算、SLAテーブル照合、SLA警告フラグ | 優先度・SLAステータス |

**チェックポイント CP-1**: トリアージサマリー（カテゴリ・優先度・SLA）を提示し、ユーザー承認を待つ。承認前に次フェーズへ進まない。

---

## Phase 2: RESPONSE（応答）

**目的**: トリアージ結果に基づく最適応答モードの実行

| ステージ | 実行条件 | 主な処理 | 出力 |
|---------|---------|---------|------|
| R1: モード選択 | 常時 | エスカレーション条件チェック（条件ベース＋感情ベース）、分析条件チェック | モード決定 |
| R2: KB検索 | 即時回答モード | KB検索（キーワード→セマンティック→カテゴリ）、信頼度スコア評価 | KB検索結果・スコア |
| R3: ドラフト生成 | 即時回答モード | 回答構造化、KB引用、パーソナライズ | 回答ドラフト |
| R4: エスカレーション引き継ぎ | エスカレーションモード | エスカレーション先選定、引き継ぎサマリー生成、暫定回答ドラフト | 引き継ぎパッケージ |
| R5: 傾向分析 | 分析モード | KPI集計、トレンド検出、レポートドラフト | 分析レポート |

**KB信頼度スコアによる分岐**:
- スコア >= 0.6 → R3（即時回答）
- スコア 0.4〜0.6 → R3「確認の上回答」テンプレート使用
- スコア < 0.4 → R4（エスカレーション）

**チェックポイント CP-2/3/4**: 各モードのドラフト/サマリー/レポートを提示し、ユーザー承認を待つ。

---

## Phase 3: QUALITY（品質）

**目的**: 全出力パスの多段階品質検証

**重要**: すべてのモード（即時回答・エスカレーション・分析）は必ずこのフェーズを通過する。バイパス禁止。

| ステージ | 実行条件 | 主な処理 | 出力 |
|---------|---------|---------|------|
| Q1: PIIスキャン | 常時（スキップ不可） | PII検出・マスキング、社内スタッフPIIも対象、検出ログ記録 | マスキング済み出力 |
| Q2: コンテンツレビュー | 常時 | KB照合、禁止表現チェック、事実確認（SLA残 < 15分の場合簡略化） | 検証済みコンテンツ |
| Q3: トーンキャリブレーション | 常時 | トーンマトリクス照合、不一致時の自動調整、Q3→R4リペアパス（最大1回） | トーン調整済み出力 |
| Q4: ディスパッチ | 常時 | CP-5最終確認（PII/精度/トーン/KB引用の全チェック通過確認）、送信、監査ログ記録 | 送信完了・監査ログ |

**リペアパス**: Q3でトーンマトリクスが「即エスカレーション」を示し、かつ現在が即時回答モードの場合 → R4へルーティング（最大1回。2回目は強制エスカレーション）

**チェックポイント CP-5**: 全チェック通過後にのみ送信実行。

---

## Phase 4: FOLLOW-UP（フォローアップ）

**目的**: 応答後の追跡・ナレッジ改善

| ステージ | 実行条件 | スキップ条件 | 主な処理 |
|---------|---------|------------|---------|
| F1: ケーストラッキング | エスカレーション済み / 確認回答送信済み / 顧客返信待ち | 即時回答で解決済み、追加接触なし | チケットステータス更新、次アクション日時記録、SLA再計算 |
| F2: FAQ候補抽出 | 同カテゴリ問い合わせ >= 5件/週 または CSAT <= 3が3件以上蓄積 | 通常範囲内 | 重複問い合わせのクラスタリング、KBギャップ特定、KB改善提案 |

---

## BUILD-TIME: PACKAGING（ビルドタイム）

**目的**: 検証済みポリシーのプラグインパッケージ化

**注意**: チケット処理ランタイムとは独立。初回ビルドまたはポリシー更新時のみ実行。

| ステージ | 処理 |
|---------|------|
| P1: プラグイン構造生成 | plugin.json / marketplace.json / エージェント定義生成 |
| P2: 自動バリデーション | 3層テスト（構造・コンテンツ・スモーク）、失敗時P1へ戻る（最大2回） |

---

## SLA早見表

| セグメント/カテゴリ | FRT目標 |
|------------------|--------|
| セキュリティインシデント（全プラン） | 1時間 |
| Enterprise | 2時間 |
| Pro | 8時間 |
| Free | 24時間 |

SLA残時間 < 15分 → SLA警告フラグ。Q2を簡略化（KB照合のみ）。
SLA超過 → 超過フラグ + マネージャー通知。

---

## ループ制御

| リペアパス | 最大回数 |
|-----------|--------|
| Q3 → R4 リペア | 1回 |
| P2 → P1 ループ | 2回 |
| 全体リトライ上限 | 3回 |

## GOOD / BAD 例

### GOOD: フェーズ遷移（正しい）

```
T3完了 → CP-1でトリアージサマリーを提示 → ユーザー承認後にR1へ進む
Q1（PIIスキャン）は SLA残 < 15分でも必ず実行する
```

### BAD: フェーズ遷移（誤り）

```
SLAが迫っているため、Q1（PIIスキャン）をスキップして即送信する
```
理由: Q1はいかなる条件でもスキップ禁止。PIIを含む出力が顧客に届く重大リスクがある。

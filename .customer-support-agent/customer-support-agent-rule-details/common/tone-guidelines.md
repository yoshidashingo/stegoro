# Tone Guidelines — トーンガイドライン

## このファイルの目的

顧客セグメント × 感情シグナルの2軸でトーンを制御するためのマトリクス・NG表現変換リスト・感情シグナル検出ルールを定義する。
`core-workflow.md` Q3（トーンキャリブレーション）で参照する。
**v2追記**: トーンマトリクスの「即エスカレーション」セルは R1 感情ベースエスカレーション条件と連動する。

---

## セグメント × 感情トーンマトリクス

| セグメント ↓ / 感情 → | normal | confused | frustrated | angry |
|---------------------|--------|----------|-----------|-------|
| **VIP** | 丁寧+迅速約束 | 丁寧+詳細説明+迅速約束 | 共感+謝罪+優先対応約束 | **即エスカレーション** |
| **General** | 標準丁寧語 | 丁寧+手順説明 | 共感+謝罪+解決策 | **即エスカレーション**（Complaint-in-progress 判定） |
| **Complaint-in-progress** | 共感+謝罪+解決策 | 共感+謝罪+詳細説明 | 共感+謝罪+解決策+フォロー約束 | **即エスカレーション** |

### 「即エスカレーション」セルの解釈（v2）

マトリクスで「即エスカレーション」と定義されたセルは、R1 Mode Selection の感情ベースエスカレーション条件と連動する二段階防衛構造を形成する。

- **R1 での処理**: R1では `angry` × `Complaint-in-progress` セグメントのみを即エスカレーション対象とする。T1で該当条件を検出した時点でエスカレーションが確定し、通常はQ3到達前に処理が確定する。
- **Q3 でのセーフティネット**: Q3はセーフティネットとして全セグメント（VIP・General・Complaint-in-progress）× `angry` をカバーする。R1で検出漏れした場合や、R1通過後に感情シグナルが `angry` と再判定された場合に Q3→R4 リペアパスで回収する（最大1回）。
- **二段階防衛構造**: R1では `angry` × `Complaint-in-progress` セグメントのみ即エスカレーション。Q3はセーフティネットとして全セグメント × `angry` をカバーする二段階防衛構造。

## トーン別文体ガイド

### 丁寧+迅速約束（VIP normal）

```
- 文末: 「〜いたします」「〜させていただきます」
- 迅速対応の明示: 「最優先で対応いたします」「本日中にご連絡いたします」
- 担当者の明示: 「担当チームが直接対応いたします」
- 敬語レベル: 最高（謙譲語・尊敬語を使い分ける）
```

### 標準丁寧語（General normal）

```
- 文末: 「〜します」「〜ください」
- 所要時間: SLA目標時間の提示
- 敬語レベル: 標準（丁寧語）
```

### 共感+謝罪+解決策（frustrated / Complaint-in-progress）

```
構造（必須）:
1. [共感] 「ご不便をおかけして大変申し訳ございません」
2. [問題認識] 「〜の件でご苦労されているとのこと、承知いたしました」
3. [謝罪] 「対応が不十分でございまして、誠に失礼いたしました」
4. [解決策] 具体的な解決手順またはエスカレーション案内
5. [フォロー約束] 「解決するまで責任を持って対応いたします」
```

### 丁寧+詳細説明（confused）

```
- 専門用語を避ける（または括弧内に説明を追加）
- 手順は番号付きリストで1ステップずつ説明
- スクリーンショット・URLを積極的に案内
- 「ご不明な点があればお気軽にご連絡ください」を末尾に追加
```

## NG表現変換リスト

| NG表現 | 変換後（推奨表現） | 理由 |
|--------|----------------|------|
| 「できません」 | 「現時点では対応が難しい状況ですが、代替案をご提案します」 | 責任放棄・否定的印象 |
| 「わかりません」 | 「確認の上、改めてご連絡いたします」 | 頼りない印象 |
| 「お客様の操作ミスです」 | 「操作手順について一緒に確認させてください」 | 顧客責任転嫁（Domain Research Pitfall 4） |
| 「マニュアルを読んでください」 | 「関連するガイドをご案内します」 | 突き放す印象 |
| 「ご不満はわかりますが」 | 「ご不便をおかけして大変申し訳ございません」 | 共感不足・対立的 |
| 「問題ありません」（確認前） | 「確認の上、問題がなければご連絡します」 | 根拠なき保証 |
| 「すぐに解決します」（確認前） | 「優先的に確認し、{時間}以内にご連絡します」 | 根拠なき約束 |
| 「たぶん〜です」 | 「確認後、正確な情報をご連絡します」 | ハルシネーション誘発（Domain Research Pitfall 1） |
| 「以上です」（締めなし） | 「ご不明な点がございましたら、お気軽にご連絡ください」 | 冷たい印象・不完全 |

## 感情シグナル検出ルール（T1）

T1チケット受付時に本文・件名をスキャンし、以下のキーワードパターンで感情シグナルを分類する。

### normal（特定シグナルなし）

検出ロジック: 以下のパターンに該当しない場合はデフォルト `normal`。

### confused（困惑）

```
検出パターン（いずれか1つ以上）:
- 「わからない」「わかりません」
- 「教えてください」「使い方」「方法を知りたい」
- 「どうすれば」「どのように」「どこから」
- 「初めて」「慣れていない」「説明してほしい」
```

### frustrated（不満）

```
検出パターン（いずれか1つ以上）:
- 「困っている」「困っています」
- 「何度も」「ずっと」「まだ」「いつまで」
- 「解決しない」「直らない」「変わらない」
- 「待っている」「返事がない」「連絡がない」
- 「なぜ」「どうして」（苦情文脈での使用）
```

### angry（怒り）

```
検出パターン（いずれか1つ以上）:
- 「ありえない」「信じられない」「最悪」「最低」
- 「解約」「退会」「やめる」（怒り文脈での使用）
- 「クレーム」「苦情」「訴える」「弁護士」「法的」
- 「詐欺」「嘘」「誇大広告」
- 「返金」（強い表現を伴う場合）
- 「怒り」「腹立たしい」「許せない」「ふざけるな」
```

**上書きルール**: キーワードが複数カテゴリに該当する場合は、より強いシグナル（angry > frustrated > confused > normal）を採用する。

## R1 感情ベースエスカレーション連携（v2）

`core-workflow.md` R1 の感情ベースエスカレーション条件:

```
感情シグナル: angry
AND
顧客セグメント: Complaint-in-progress
→ 即エスカレーション（R4）
```

本ファイルのトーンマトリクスでの対応セル: `VIP × angry`、`General × angry`、`Complaint-in-progress × angry` の全3セルが「即エスカレーション」。

R1では `angry` × `Complaint-in-progress` セグメントのみ即エスカレーション。Q3はセーフティネットとして全セグメント × `angry` をカバーする二段階防衛構造。R1で正しく検出された案件はQ3に到達しない。Q3は検出漏れ時および全セグメントの最終安全装置として機能する。

## GOOD / BAD 例

### GOOD: frustrated 顧客への適切なトーン（正しい）

```
田中様

ご不便をおかけして大変申し訳ございません。
何度もご連絡いただいているにもかかわらず、解決に至らず誠に失礼いたしました。

本件を最優先で対応いたします。以下の手順をお試しください。
[解決策]

本日中に進捗をご連絡いたします。
```

### BAD: frustrated 顧客への不適切なトーン（誤り）

```
田中様

お問い合わせありがとうございます。
設定メニューから変更してください。
```
理由: Domain Research Pitfall 4（トーンミスマッチ）。共感・謝罪がなく、frustrated の顧客のクレームを激化させる。

---

### GOOD: angry 顧客の検出とエスカレーション（正しい）

```
T1で「解約する」「最悪」のキーワードを検出 → emotion_signal = angry
顧客セグメント = Complaint-in-progress
→ R1で感情ベースエスカレーション条件に該当 → 即座にR4へ
```

### BAD: angry シグナルの無視（誤り）

```
「解約する」というキーワードを見たが、KB検索で回答が見つかったため
emotion_signal を normal のまま即時回答モードで処理した。
```
理由: Domain Research Pitfall 3（エスカレーション判定ミス）+ Pitfall 4（トーンミスマッチ）。
感情ベースエスカレーション条件を見落とし、怒っている顧客に事務的な回答を送信するリスク。

# コマンド: validate-policy（ポリシー品質検証）

## 概要

カスタマーサポートエージェントのステアリングポリシー一式に対してP2（Automated Validation）の3層テスト（構造テスト・コンテンツテスト・スモークテスト）を実行するコマンド。ポリシー更新後や初期構築後の品質確認に使用する。

## 使用方法

```
/validate-policy [オプション]
```

## オプション

| オプション | 説明 |
|-----------|------|
| `--layer1` | 構造テストのみ実行（ファイル存在・相互参照・Markdown妥当性） |
| `--layer2` | コンテンツテストのみ実行（Dim12-15の品質次元検証） |
| `--layer3` | スモークテストのみ実行（9フロー到達可能性検証） |
| オプションなし | 全3層を順番に実行 |

## 検証内容

### Layer 1: 構造テスト

- 29ファイル（core 1 + common 12 + phase rules 16）の存在確認
- `core-workflow.md` 内の全参照先ファイルの解決確認
- Markdown見出し階層の妥当性（`##`→`###` の飛びなし）
- フローパスのデッドエンド検出

### Layer 2: コンテンツテスト

- **Dim 12**: ドメイン特化率（各ファイルで40%以上）
- **Dim 13**: GOOD/BAD例カバレッジ（各ファイルで2件以上）
- **Dim 14**: アーティファクトテンプレート完備率（6ファイルで100%）
- **Dim 15**: 4 Pitfalls引用率（エラーハンドリングの50%以上）

### Layer 3: スモークテスト（9フロー）

1. 即時回答フロー
2. エスカレーションフロー（条件ベース）
3. エスカレーションフロー（KB未ヒット）
4. エスカレーションフロー（Q3リペアパス）
5. 分析フロー
6. フォローアップフロー
7. SLA超過パス
8. PACKAGINGフロー
9. Hybrid切り替え（5パターン全分岐）

## 出力

検証完了後、以下を出力する。

```
[P2 VALIDATION REPORT]
総合判定: PASS | FAIL
- Layer 1（構造）: PASS | FAIL
- Layer 2（コンテンツ）: PASS | FAIL
- Layer 3（スモーク 9フロー）: PASS | FAIL
```

FAILの場合は失敗箇所の詳細とP1での修正指示を合わせて出力する。

## 注意事項

- FAILが継続する場合、P2→P1ループは最大2回まで自動修正を試みる
- 2回のループでもFAILが続く場合は手動レビューが必要
- ポリシー更新後は必ずこのコマンドで検証してからリリースすること

# コマンド: new-ticket（新規チケット処理開始）

## 概要

新規メール問い合わせのチケット処理を開始するコマンド。メール本文を受け取り、T1（チケット受付）からワークフローを起動する。

## 使用方法

```
/new-ticket [メール本文]
```

## 引数

| 引数 | 必須 | 説明 |
|------|------|------|
| メール本文 | 必須 | 顧客からのメール全文（件名・本文・差出人情報を含む） |

## 処理フロー

1. **T1（チケット受付）**: チケットID生成・PIIフラグ付与・感情シグナル検出・重複チェック
2. **T2（分類）**: カテゴリ/サブカテゴリの自動判定
3. **T3（優先度評価）**: P1〜P4の優先度スコアリングとSLA割り当て
4. **R1（モード選択）**: 即時回答/エスカレーション/分析の3モードから選択
5. 以降、選択されたモードに応じてワークフローを継続

## 実行例

```
/new-ticket
件名: パスワードリセットができません
差出人: customer@example.com
本文:
ログインしようとしたところ、パスワードリセットのメールが届きません。
何度試しても届かないので困っています。
```

## 注意事項

- メール本文にPIIが含まれる場合、T1でフラグが付与される（送信前にQ1でマスキング）
- 同一顧客からの問い合わせが24時間以内に存在する場合、重複チェックが実行される
- SLAは顧客プランに基づいて自動割り当てされる
